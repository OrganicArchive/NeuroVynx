# Security Policy

## Supported Versions

Security updates are provided for the following versions:

| Version | Supported |
| ------- | --------- |
| Latest release | ✅ |
| Older releases | ❌ |

We currently only support the latest release. Older releases may not receive security fixes.

## Reporting a Vulnerability

We take the security of NeuroVynx seriously. If you discover a security vulnerability, we encourage you to report it privately to ensure the safety of users, research workflows, and processed data.

### Preferred Reporting Paths

1. **Private Reporting**: The preferred method for reporting a vulnerability is through [GitHub Private Vulnerability Reporting](https://github.com/OrganicArchive/NeuroVynx/security/advisories/new) (if enabled for this repository).
2. **Direct Contact**: If private reporting is unavailable, please contact the repository maintainer directly via the maintainer's GitHub profile, or by opening a non-sensitive general issue requesting private contact.

**Do NOT open a public issue for security vulnerabilities.**

### Scope of Reportable Issues

NeuroVynx is a research-oriented, non-diagnostic platform. Security reports that affect integrity, reliability, or safe processing of research data are also welcome.

We are particularly interested in reports concerning:
- **Data Ingestion**: Vulnerabilities in data ingestion and parsing (EDF/BDF signals).
- **Dependency Vulnerabilities**: Security flaws in our core scientific or platform dependencies.
- **Unsafe Input Handling**: Potential exploits in data processing or API endpoints.
- **Local Execution**: Security concerns regarding the local Electron/Node/Python environment.

### Response Expectation

- **Review**: We provide a best-effort review of all legitimate security reports.
- **Acknowledgment**: We will acknowledge receipt of your report as soon as possible.
- **Disclosure**: We follow responsible disclosure practices and ask that you give us reasonable time to address the issue before publicizing it.

Thank you for helping keep NeuroVynx secure.
